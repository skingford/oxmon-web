## ADDED Requirements

### Requirement: 页面必须覆盖四类异步状态
所有接口驱动页面 MUST 统一覆盖 loading、error、empty、success 四类异步状态，并确保状态切换可预测。

#### Scenario: 数据请求返回不同结果
- **WHEN** 页面请求分别处于加载中、失败、空结果与成功结果
- **THEN** 页面 SHALL 显示对应状态视图且不出现状态缺失

### Requirement: 错误状态必须提供恢复动作
当页面进入错误状态时，系统 MUST 提供至少一种可执行恢复动作（例如重试或刷新），并在恢复成功后返回正常展示状态。

#### Scenario: 请求失败后用户重试
- **WHEN** 用户在错误状态触发重试动作且接口恢复正常
- **THEN** 页面 SHALL 从错误状态恢复到成功或空结果状态

### Requirement: 空状态需提供下一步引导
当接口返回空数据时，系统 SHOULD 提供明确的下一步引导或解释信息，避免用户停留在无反馈状态。

#### Scenario: 页面首次进入空状态
- **WHEN** 页面请求成功但无可展示数据
- **THEN** 页面 SHALL 显示空状态说明并提供可执行引导动作或路径提示
